🕷️ Hexapod GUI Code Reference

Complete Class & Method Documentation

📋 Code Structure Overview

🏗️ Architecture

  • 6 Main Classes - Core application components
  • Threading - Non-blocking UI operations
  • Qt Signals - Safe inter-thread communication
  • Network Layer - SSH + TCP socket communication

🎯 Key Technologies

  • PyQt5 - Modern GUI framework
  • OpenCV - Computer vision processing
  • DeepFace - AI face recognition
  • Paramiko - SSH client automation

🔄 Data Flow

  • UI Events → Commands → Network
  • Video Stream → CV Processing → Display
  • AI Recognition → Results → Logging
  • Sensor Data → Response → UI Update
🤖 HexapodGUI (Main Application Class)
Main window class that inherits from QMainWindow. Orchestrates the entire application, manages UI components, handles user interactions, and coordinates all subsystems including video streaming, face recognition, and robot control.
Method Type Parameters Purpose
__init__ Constructor self Initialize main window, set up variables, create UI components, and start server status check
init_ui UI Setup self Create the main UI layout with left control panel and right video panel, apply modern styling
create_control_panel UI Creation self Build the left panel with connection controls, movement buttons, sensor cards, and action buttons
create_video_panel UI Creation self Create the right panel containing video display area and activity log text box
one_click_connect Connection self Start connection thread to SSH into Pi, launch server processes, and establish communication
check_server_status Network Check self Test if the server is already running by attempting TCP connection to Pi
send_command Robot Control self, command Send JSON-formatted commands to Pi via TCP socket and process responses
handle_response Data Processing self, response, command Parse server responses and update UI elements (distance, battery, status cards)
start_video_stream Video Control self Launch video server on Pi and start local video capture thread for live streaming
stop_video_stream Video Control self Stop video streaming, clean up video server processes, and reset video display
start_face_recognition AI Control self Initialize DeepFace, create worker thread, start video server, and begin face recognition
stop_face_recognition AI Control self Safely stop recognition thread, clean up resources, and reset UI state
update_connection_status UI Update self, message, color Update connection status display with colored messages and log activity
on_connection_ready Event Handler self, ready Enable/disable all control buttons based on server connection status
log_activity Logging self, message Add timestamped messages to activity log and auto-scroll to bottom
closeEvent Event Handler self, event Handle application shutdown: stop threads, clean up resources, close connections
🧠 FaceRecognitionWorker (AI Processing Thread)
QObject-based worker class that runs in a separate QThread. Handles real-time face detection using Haar Cascades and face recognition using DeepFace AI model. Processes video frames and emits Qt signals for thread-safe UI updates.
Method Type Parameters Purpose
__init__ Constructor self, rpi_host Initialize worker with Pi host address, set up thread control variables and mutex
start_recognition AI Processing self Main processing loop: capture frames, detect faces, run DeepFace recognition, emit results
stop_recognition Thread Control self Thread-safe method to stop the recognition loop using mutex protection
🔌 ServerConnectionThread (Network Management)
QThread subclass responsible for all network operations. Handles SSH connections to Raspberry Pi, manages server processes, monitors connection status, and provides automatic server startup capabilities.
Method Type Parameters Purpose
__init__ Constructor self, action="check" Initialize thread with action type (connect/check) for different operation modes
run Thread Execution self Main thread execution method that calls connect or check based on action type
connect_ssh SSH Connection self Establish SSH connection to Pi using Paramiko with error handling and timeout
connect_and_start_server Server Management self SSH to Pi, check if server running, start if needed, verify connection works
check_server_status Status Check self Test TCP connection to server without starting it, report status via signals

🔄 How It All Works Together

🚀 Startup Sequence

  • main() creates Qt app and splash screen
  • HexapodGUI.__init__() initializes all components
  • init_ui() creates the visual interface
  • check_server_status() tests Pi connection

🌐 Connection Flow

  • one_click_connect() starts connection thread
  • ServerConnectionThread handles SSH operations
  • connect_and_start_server() launches Pi processes
  • on_connection_ready() enables controls

🎮 Control Operations

  • Button clicks trigger command functions
  • send_command() creates TCP connection
  • JSON messages sent to Pi server
  • handle_response() updates UI elements

🧠 AI Recognition Flow

  • start_face_recognition() creates worker thread
  • FaceRecognitionWorker processes video frames
  • DeepFace.find() performs AI recognition
  • Qt signals update main UI safely
